## 1. PURPOSE & BOUNDARY CONDITIONS

This playbook exists because Compilore's standalone subscription cost 
(~1,200–1,800 PLN/msc Team tier) cannot be funded as a standalone EU 
grant — too small to clear minimum project thresholds. However, when 
Compilore is bundled into a broader digital transformation project on 
the client side, 50–60% of the full project (including Compilore 
license, integration, hardware, training) can become non-repayable 
subsidy.

**Critical boundary:** The grant applicant is **always the client 
(e.g. HermesTools), never Headframe Sp. z o.o.** Headframe is a 
technical vendor providing:
- Pre-paid multi-year license (structured as capital-izable WNiP)
- Integration services invoiced as part of the project budget
- Technical documentation supporting the "innovation" scoring criteria
- Letter of technical fit as needed

**Not applicable paths:**
- Headframe applying for R&D grants on its own (pre-revenue, too small)
- Compilore as product-level grant (not eligible under FENG 2.21 Dig.IT 
  without manufacturing PKD on vendor side)
- Consortium applications with client on R&D tracks (Ścieżka SMART) — 
  unless a genuine R&D component is added (not currently on roadmap)

## 2. DECISION TREE: Should This Client Pursue A Grant?

Apply this sequence before proposing a grant path to any prospect:

1. **Has the pilot (or equivalent demonstration) succeeded?** Pilot 
   metrics must hit success gate from 09_PILOT.md: 80%+ citation 
   accuracy, champion recommendation. If no — DO NOT propose grant, 
   no narrative credibility.
2. **Is the client's PKD section C (manufacturing) or do they have 
   genuine "usługi produkcyjne" revenue >30%?** If yes — Dig.IT 
   is an option. If no — Dig.IT is closed.
3. **Is the client in Śląskie Voivodeship and ≥3 years old?** If yes 
   — FE Śląskie 1.8 is an option. Other voivodeships have parallel 
   instruments — check regional FE program.
4. **Does the client's revenue support 40–50% own contribution?** 
   Grants cover 50–60% max. Client must fund the rest. Grant 
   programs verify this via financials.
5. **Does the client have management bandwidth for 8–9 month 
   application lifecycle?** Most grant failures are preventable 
   (incomplete application, missed deadlines). Weak commitment = 
   skip the path.

If all 5 answers are yes: propose. Otherwise: defer or skip.

## 4. FE ŚLĄSKIE 1.8 — DETAILED PLAYBOOK

### Eligibility check for HermesTools

| Criterion | Requirement | HermesTools status |
|---|---|---|
| MŚP status | micro/small/medium | Need headcount confirmation (likely medium 30–250) |
| Geographic | Śląskie Voivodeship activity | ✅ Bielsko-Biała HQ |
| Minimum operating history | 3+ completed financial years | ✅ 21+ years as sp. z o.o. |
| Sectoral scope | Trading firms admissible (manufacturing preferenced) | ✅ admissible, scores lower |
| Own contribution capacity | ~40% of project value | ❓ Pending financial discussion |
| No overlapping aid on same costs | DNSH + aid rules | Must verify case-by-case |

### Eligible costs (verify with latest regulamin at application time)
- Fixed assets (środki trwałe): servers, workstations, scanners, specialized hardware
- Intangible assets (WNiP): software licenses as multi-year purchase or capitalized
- Specialized training
- Professional services supporting implementation
- Cybersecurity enhancements

**Compilore structuring:** 3-year pre-paid license as WNiP (capital-ized 
intangible), NOT as monthly OpEx subscription. Standard SaaS monthly 
billing is problematic for grant eligibility. Draft license agreement 
must reflect this.

### Minimum viable project structure (500k+ PLN)

```
Total project:                   ~650k PLN
Own contribution (40%):          ~260k PLN
Subsidy (60%):                   ~390k PLN

Cost breakdown:
- Compilore license 3-year pre-paid (WNiP):       150k PLN
- ERP/CRM integration work (vendor services):     180k PLN
- Hardware (workstations, scanners, servers):     120k PLN
- Training + change management:                    80k PLN
- Consulting + grant writing (optional):           50k PLN
- Contingency:                                     70k PLN
```

### Application timeline (from decision to first capital flow)

- Month 0: decision to pursue + grant writing engagement
- Month 1–2: application preparation (technical docs, financial 
  projections, DNSH analysis, digital maturity test)
- Month 2 (latest): submission
- Month 3–5: formal evaluation (legally mandated 80–110 days)
- Month 5–6: decision + contract negotiation
- Month 6–7: collateral + first disbursement
- **Total: 7–9 months from decision to cashflow**

### Scoring leverage points specific to HermesTools

- Calibration services as "technological necessity" (ISO 6789 compliance 
  pressure on clients)
- Aerospace clients (Recoules product line) as "high-stakes regulatory 
  environment"
- Multi-country operations (PL/CZ/SK) as "regional economic impact"
- Export orientation as "increased competitiveness" criterion
- Post-IATF 16949 revision 2025 as "external regulatory driver" 
  justifying transformation urgency

### Weakness factors to mitigate

- PKD 46.14.Z in section G (trade) — mitigation: emphasize calibration 
  and assembly station design as dominant revenue streams if true; 
  consider adding section C PKD as secondary activity before application
- "Nice to have" AI risk — mitigation: lead with quantified advisor 
  time savings from pilot data
- Non-production status — mitigation: frame as "industrial services 
  for manufacturing ecosystem"

## 5. DIG.IT — CONDITIONAL PLAYBOOK

**Only pursue if HermesTools revenue from calibration + assembly station 
design + integration work exceeds 30% of total revenue.**

If that condition holds, Dig.IT is smaller, simpler, faster than 
FE Śląskie 1.8:
- Grant range: 150k–850k PLN
- Subsidy: up to 50% (de minimis)
- Next nabór: June 2026 (confirm at digit.arp.pl)
- 5-year operating history required — satisfied

Application structure similar to FE Śląskie 1.8 but with de minimis 
aid constraints (current de minimis usage by client must be tracked).

**If revenue share falls short — skip this path.** ARP is strict on 
sectoral eligibility; formal rejection at intake is the standard 
outcome for section G trading firms.

## 6. TRANSFORMATION TRIGGER NARRATIVES

The Gemini-provided "INTOOL merger" trigger is fabricated (see DR-13 
RESULTS in 08_RESEARCH.md). Use one of these instead:

### Option A — Regulatory documentation pressure (safe default)
IATF 16949 revision 2025, AS9100→IA9100 transition 2026–2027, F-Gas 
EU 2024/573 torque calibration archiving requirements, Industry 4.0 
telemetry mandates. HermesTools' end-clients face escalating 
documentation demands; existing manual catalog search cannot scale 
without structural digital transformation.

### Option B — Institutional knowledge capture (if demographics fit)
30+ year company, senior technical advisors approaching retirement. 
Compilore as enterprise institutional memory. **Requires:** 
verification of actual advisor age distribution. If average age 
>50, this narrative is authentic. If average age <40, skip.

### Option C — ERP modernization prep (strongest narrative — depends on D-84)
10-year-old custom ERP is data silo. Compilore as intelligence layer 
enables eventual ERP modernization without full replacement. Best 
fit for FE Śląskie 1.8 "zasadnicza zmiana procesu biznesowego" 
requirement. **Requires D-84 pilot success gate hit** — only viable 
if disclosed to HermesTools management.

### Selection rule
- If pilot succeeds and D-84 disclosure triggered → Option C (strongest)
- If pilot succeeds but D-84 stays private → Option A (safe)
- If pilot hits B-specific triggers (retirement narrative lands) → Option B

## 8. GTM POSITIONING — "WE HELP YOU GET A GRANT FOR THIS"

This playbook reframes the Compilore sales conversation from 
"buy our tool" to "buy our tool with 60% off through an EU grant we 
help you secure." This is a material GTM advantage in Polish SME 
sales (per DR-11 conservative-buyer research).

**Sales sequence with grant angle:**
1. Pilot demonstration → proves tool works
2. ROI discussion with champion → quantified value
3. Grant path briefing with management → reduces sticker price 40–60%
4. Referral to grant consultant or Bartek-guided application
5. Contract signed contingent on grant approval (dual-contract mechanism: 
   small PoC funded by client + master agreement triggered on grant 
   disbursement)

**Risk allocation:** If grant fails after 8 months, client has not 
wasted large upfront capital (only PoC). Vendor has kept relationship 
active through pilot + PoC fees.
